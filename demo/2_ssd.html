Detection with SSDÂ¶

In this Demo, we convert a ssd300-vgg16 trained by Caffe to bmodel for image classfication, while both fp32 bmodel and int8 bmodel are used. We implemented five cases.

Among the first four cases, we use different decoding and preprocessing libs. Except opencv, which is the public released version of opencv, other libs like bm-opencv, bm-ffmpeg and bmcv are all bitmain versions, which exploiting TPU to decode/process videos/images.

0 video image opencv opencv fp32 int8 ssd300-vgg16 static 1 1 N
1 video image bm-ffmpeg bmcv fp32 int8 ssd300-vgg16 static 1 1 N
2 video image bm-ffmpeg bmcv fp32 int8 ssd300-vgg16 static 1 4 N
3 video image bm-opencv bm-opencv fp32 int8 ssd300-vgg16 static 1 1 N
4 video image bm-opencv bmcv fp32 int8 ssd300-vgg16 static 1 1 N
